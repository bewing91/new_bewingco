SELECTED SCREENS

An engaging and transparent application process

RESEARCH

Understanding the current application process

Current Flow: Too many steps in too many places

The current teacher application process moved prospective teachers between various screens on the Outschool website, emails, and third-party sites. The process felt disjointed and confusing.

Why are prospective teachers dropping off?

Over the past few months, the Outschool Product team collected feedback from teachers and prospective teachers about the application process. After reviewing this feedback, we identified 3 key pain points prospective teachers were experiencing:

Pain Point 1:
Lack of transparency
Users didn't know where they were in the process or when they would get a response.
Pain Point 2:
Disparate process
Users didn't understand how or where to continue to the next step of the application process.
Pain Point 3:
Wasted excitement
Users were excited to engage with the platform but had no way to while they were waiting.
EXPLORING SOLUTIONS

How might we make it easy for teachers to understand their application status and stay engaged while they wait?

New Flow: One main dashboard to guide teachers through the application process
After exploring a number of options, we decided that a dashboard would be the best solution to the issues applicants were experiencing. A dashboard would give the applicant:

  • Application status in a consistent place
  • Guiding information about next steps
  • Resources to engage with while waiting
While we explored embedding the background check to further simplify the flow, we ultimately decided an embedded form would complicated the interface and potentially hurt the user experience.

Version 1: Status visualization and resources

To provide transparency, we created a status section at the top of the dashboard with an arrow visualization to make it easy for applicants to see where they were in the process. The copy, CTA, and status change based on the applicant's stage.

To increase engagement, we created a resources section below the status where users can learn about Outschool and stay engaged while they wait. The colors were chosen to reflect a design pattern elsewhere in the Outschool site.

Version 2: A timeline and more interactivity

Usability testing takeaways
After a round of usability testing, we learned that users:
  • Found the arrow visualization confusing and didn't understand what stage they were in.
  • Liked the resources provided but wanted to be able to take action based on their learnings.
Resulting improvements
To improve clarity and better match an existing Outschool design pattern, we swapped the arrows out for a timeline. For greater interactivity, we added cards to enable applicants to make changes to the class they created earlier in the application process, and to schedule a call with an existing teacher.

Mobile

Final Version: Emphasized focal point

Given that our primary goal was to reduce dropoff, and the status section is what guides applicants through the process, we decided that we needed to further emphasize it. To draw further attention to the status, we removed the colored backgrounds from the resource cards and increased the contrast of the status section by changing the background to white. We swapped out the illustrations to add back some visual interest to resources and maintain visual consistency with other parts of the site.

Status states
State 1: Application submitted
The applicant first lands on this page when they submit their application. They're in this state until the Outschool team approves it.
State 2: Application approved
Once their application is approved, they need to complete a background check. This state is focused on encouraging them to start it.
State 3: Background check started
Background checks can take a few days. Once they've submitted their background check, they need to wait for it to be completed before they can become an Outschool teacher.
State 4: Background check complete
When the check is complete, the user will be automatically taken to their teacher dashboard when they next visit Outschool. They'll only see the application dashboard in this state if they visit its URL directly.
OUTCOME

Implementation

Throughout the design process, we worked with the engineering and operations teams to ensure that the final design would be relatively easy to implement and would make an outsized impact on the Outschool experience. Implementation of the new application status dashboard is planned for Q4 2019.

What's Next

Outschool runs its team on bi-monthly cycles and as a result, the timeline for design of the dashboard was fairly tight. With more time, I would rethink the entire application process to make it a smoother, more welcoming experience. Currently, the applicant needs to fill out the application and then goes through the additional steps found in this dashboard. There's ample room to improve the user experience by further integrating this dashboard with the application itself.
